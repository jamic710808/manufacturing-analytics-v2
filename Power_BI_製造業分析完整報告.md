## 一、整體架構概覽

### 分析主線脈絡

```
原料採購 → 庫存管理 → 生產投料 → 成品產出 → 成本核算
    ↓            ↓           ↓          ↓           ↓
供應商分析   庫存分析    BOM扁平化   良率監控   成本差異追蹤
```

### 三大分析域 × 七個核心報告頁

| 分析域 | 核心問題 | 關鍵指標 |
|--------|---------|---------|
| **庫存分析** | 庫存結構是否健康？資金是否被呆滯料凍結？ | 期末庫存餘額、周轉天數、ABC分類、動銷率 |
| **供應商分析** | 採購成本是否可控？供應商交期是否穩定？ | 帕雷托集中度、價格差異率、遲交率、採購周期 |
| **成本分析** | 生產成本結構如何？標準 vs 實際差在哪？ | BOM扁平化、良率、料工費結構、單位成本差異 |

## 二、資料模型設計

### 星型模式 (Star Schema) 配置

#### 維度表 (Dimension Tables / D 前綴)
| 表名 | 說明 | 關鍵欄位 |
|------|------|---------|
| `D日期表` | 主日期維度，供所有事實表連結 | Date, Year, Quarter, Month, YearMonth, WeekNum |
| `D日期辅助表` | 複製自主日期表，用於趨勢圖 X 軸（避免切片器干擾） | Date（與 D日期表 無關聯） |
| `D成品档案` | 成品主檔，SKU 主索引 | 存貨編碼、存貨名稱 |
| `D原材料档案` | 原料主檔，物料分類依據 | 存貨編碼、一級分類、二級分類 |
| `D采购订单号` | 採購訂單維度（UNION 去重建立） | 訂單編號 |
| `D合并物料表` | 成品+原料合并維度（UNION 建立） | 存貨編碼、名稱、分類層級 |
| `D成本类型` | 費用類型三級層級維度 | 一級成本類型→二級物料→三級費用名稱 |

#### 事實表 (Fact Tables / F 前綴)
| 表名 | 說明 | 關鍵欄位 |
|------|------|---------|
| `F成品库存明细` | 成品庫存每日明細 | 日期、存貨編碼、結存金額 |
| `F原材料库存明细` | 原料庫存每日明細 | 日期、存貨編碼、結存金額 |
| `F采购到货单` | 採購到貨明細（供應商價格分析用） | 供應商、訂單號、存貨編碼、原幣單價、數量、金額 |
| `F采购订单` | 採購訂單（含計劃交期） | 訂單編號、日期、計劃到貨日 |
| `F生产成本明细` | 生產成本每工單明細（料工費） | 存貨編碼、訂單號、費用編碼、單價、數量、金額 |
| `F原材料采购到货单列表` | 採購供應商彙總清單 | 供應商、品名、金額 |

#### 關鍵建模決策
- **主日期表 vs 日期辅助表**：兩表無關聯，辅助表專供趨勢折線圖 X 軸，避免切片器篩選污染時間軸
- **D采购订单号**：使用 `UNION + FILTER` 對兩個來源表去重合并，確保採購周期計算不重複
- **D合并物料表**：使用 `UNION` 將成品與原料主檔合并，統一物料層級結構

## 三、庫存分析模組

### 3.1 核心分析架構

```
庫存效能分析                    庫存風險分析
├── 期末庫存餘額（原料+成品）       ├── 超期庫存（>90天）= 呆滯料
├── ABC 分類（金額帕雷托）          ├── 專用料 vs 通用料 vs 非生產用料
├── Top 10 高價值庫存              └── 資金佔用監控
└── SKU 動銷分析
    ├── 在檔 SKU 數
    ├── 在售 SKU 數（有出庫）
    ├── 動銷率 = 在售/在檔
    └── 近三月在庫動銷率
```

### 3.2 原料分類邏輯

| 類型 | 判斷條件 | 業務意義 |
|------|---------|---------|
| **專用料** | 該原料生產的成品種類 ≤ 5種 | 高客製化，斷料風險高 |
| **通用料** | 生產成品種類 > 5種 | 共用原料，可彈性調配 |
| **非生產用料** | 與任何成品 BOM 無關 | 非直接生產消耗品（如辦公用品、維修備件） |

### 3.3 關鍵因素

1. **期末庫存計算邏輯**：必須抓「最後一天」的庫存快照，而非求和（避免重複計算）
2. **日期輔助表設計**：趨勢圖必須用輔助表 X 軸，否則切片器會干擾歷史趨勢顯示
3. **動銷率分母選擇**：「近三月在庫動銷率」比「全檔動銷率」更能反映實際流動狀況
4. **ABC 分類臨界**：根據企業實際分布動態調整，不硬套 80/20

## 四、供應商分析模組（價格）

### 4.1 核心分析架構

```
採購成本控制
├── 帕雷托分析（80/20）
│   ├── 供應商採購金額排名
│   └── 累計佔比折線（識別核心供應商集中度）
├── 價格差異分析
│   ├── 最高/最低採購單價
│   ├── 價格差異率 = (最高-最低)/最低
│   └── 價格趨勢 = (最新-平均)/平均
└── 歷史採購趨勢（同供應商同品項）
```

### 4.2 關鍵因素

1. **去重計算單價**：同一訂單多次到貨需先在 SUMMARIZE 層面去重，再取 MIN/AVG
2. **帕雷托可視化**：主軸為採購金額長條，次軸為累計佔比折線，兩軸分開設置
3. **散佈圖**：X 軸為訂購量，Y 軸為遲交天數，可視化識別「量大但頻繁遲交」的問題供應商
4. **多維度雷達圖**：價格、交期、品質、服務四維度綜合評分，替代單一指標排名

## 五、供應商分析模組（採購時效）

### 5.1 核心分析架構

```
採購時效分析
├── 採購周期計算
│   ├── 最早到貨天數（DATEDIFF 下單→首次到貨）
│   ├── 計劃天數（訂單計劃交期）
│   └── 平均採購周期（AVERAGEX 對每張訂單）
├── 遲交分析
│   ├── 遲交次數 / 總下單次數 = 遲交率
│   ├── 早於計劃次數
│   └── 平均遲交天數
└── 供應商排行（遲交天數 Top N）
```

### 5.2 關鍵因素

1. **訂單維度表建立**：必須 UNION 兩個來源（採購訂單 + 到貨單）去重，確保一張訂單只計一次
2. **周期計算前提**：先用 CALCULATE 取得「下單日期」，再 DATEDIFF 到「最早到貨日」
3. **遲交判斷邏輯**：實際最早到貨天數 > 計劃天數 → 遲交，反之為準時或提前
4. **ADDCOLUMNS 優化**：計算「平均遲交天數」時先用 ADDCOLUMNS 附加差異欄位，再 AVERAGEX

## 六、成本分析模組：BOM 扁平化

### 6.1 BOM 扁平化必要性

```
標準 BOM 結構（多層）:
成品 A
  └─ 半成品 B（虛擬件）
        ├─ 原料 C
        └─ 原料 D

扁平化後（Power BI 分析用）:
成品 A → 原料 C（用量 = BOM用量 × 半成品B展開係數）
成品 A → 原料 D（用量 = BOM用量 × 半成品B展開係數）
```

### 6.2 Power Query 扁平化步驟

1. **篩選虛擬半成品**：母件編碼以 "4" 開頭 → `Table.SelectRows(source, each Text.StartsWith([母件编码], "4"))`
2. **保留最新版本**：Group By 母件編碼，取 `List.Max([版本代号])`
3. **自身合并**：母件編碼 = 子件編碼，取得下一層 BOM 結構
4. **計算展開用量**：`基本用量_扁平 = [基本用量] × [合并1.基本用量]`
5. **UNION 合并**：將扁平化後的虛擬件行與直接原料行 UNION，形成完整扁平 BOM

### 6.3 關鍵因素

1. **虛擬半成品識別**：依企業 ERP 編碼規則（如首碼為 "4"）篩選，需與 IT/ERP 確認
2. **版本管理**：BOM 存在多版本時，只取最新版，避免舊版干擾計算
3. **用量累積**：多層展開時，用量需逐層相乘（不是相加）
4. **UNION 後去重**：原始直接原料行 UNION 展開行後，可能出現重複，需確認主鍵

## 七、成本分析模組：生產成本報告

### 7.1 核心分析架構

```
生產成本報告
├── 總覽 KPI
│   ├── 實際消耗金額（本期 vs 上期）
│   ├── 單位生產成本（元/母件單位）
│   └── 生產種類數（DISTINCTCOUNT）
├── 母件矩陣（按產品展開）
│   ├── 母件名稱/編碼
│   ├── 產量（去重後 SUMX）
│   ├── 子件消耗金額
│   ├── 單位成本
│   └── 母件單位（SELECTEDVALUE）
├── 趨勢分析（近12個月）
│   ├── 產量趨勢折線（日期輔助表 X 軸）
│   └── 單位成本趨勢折線（雙Y軸）
└── 成本結構
    ├── 環形圖（材料/製造/人工 三大類）
    ├── 柱形圖（料工費明細）
    └── 材料成本明細（按二級分類）
```

### 7.2 產量去重邏輯

生產成本明細表中，同一工單的原料子件會有多行記錄，若直接 SUM 產量欄位會導致重複計算。

**正確做法**：先 SUMMARIZE 到訂單粒度，再 SUMX

```dax
母件产量 =
VAR T = SUMMARIZE(
    'F生产成本明细',
    [存货编码], [订单号], [订单行号], [产量]
)
RETURN SUMX(T, [产量])
```

### 7.3 業務邏輯：規模效應驗證

| 場景 | 產量 | 單位成本 | 判斷 |
|------|------|---------|------|
| 正常 | ↑ | ↓ | 規模效應正常運作 |
| 異常 | ↑ | ↑ | 需排查：物料漲價？良率下降？ |
| 異常 | ↓ | ↓ | 需排查：成本計算是否有誤？ |

### 7.4 關鍵因素

1. **單位換算**：子件以 KG 計，母件以「包」計，單位成本需乘以倍率換算為元
2. **日期輔助表**：趨勢折線 X 軸必須使用輔助表，否則切片器選月份後趨勢圖會消失
3. **SELECTEDVALUE 母件單位**：在筛選上下文中確保每個產品只對應唯一單位
4. **雙Y軸設計**：產量（值域大）用主Y軸，單位成本（值域小）用輔助Y軸

## 八、成本差異分析模組

### 8.1 兩種差異分析方法

| 方法 | 適用場景 | 計算方式 |
|------|---------|---------|
| **環比差異** (MoM) | 自產自銷企業、產品迭代慢 | 本期值 - 上期值 |
| **同比差異** (YoY) | 季節性業務、多年數據穩定 | SAMEPERIODLASTYEAR 函數 |
| **不適合同比** | OEM/ODM 代工企業 | 產品每年更換，無同期數據可比 |

### 8.2 差異分析框架

```
成本差異分析
├── 產量差異（Environment Level）
│   ├── 差異額 = 本期產量 - 上期產量
│   └── 差異率 = 差異額 / 上期產量
├── 單位生產成本差異（Product Level）
│   ├── 環比差異率
│   ├── 絕對值（ABS）→ 異動提醒分級
│   └── 條件格式圖標（旗幟分三級）
├── 量差分析（Material Quantity Variance）
│   └── 子件每單位用量 = 子件消耗數量 / 母件產量
└── 價差分析（Material Price Variance）
    └── 子件單位金額 = 實際消耗金額 / 實際消耗數量
```

### 8.3 異動預警分級

| 異動幅度 | 圖標 | 管理動作 |
|---------|------|---------|
| < 1% | 無圖標 | 正常誤差，忽略 |
| 1% ~ 2% | 半填充黃旗 | 注意觀察 |
| 2% ~ 5% | 全填充黃旗 | 需分析原因 |
| > 5% | 紅旗 | 立即處理，上報高管 |

### 8.4 TREATAS 跨模型篩選

當合并物料表無法直接關聯採購訂單表時（無關係路徑），使用 TREATAS 建立臨時篩選：

```dax
採購趨勢 =
VAR X = VALUES('D日期辅助表'[Date])
VAR Y = DATESINPERIOD('D日期表'[Date], MAX('D日期表'[Date]), -12, MONTH)
RETURN
CALCULATE(
    [平均采购价格],
    TREATAS(VALUES('D合并物料表'[存货编码]), '原材料档案'[存货编码]),
    INTERSECT(X, Y)
)
```

## 九、完整 DAX 公式彙整與解釋

### 9.1 日期表建立

```dax
-- 基礎日期表（Power Query 建立後，DAX 補充計算欄位）
Year        = YEAR([Date])
Quarter     = QUARTER([Date])
Month       = MONTH([Date])
YearMonth   = YEAR([Date]) * 100 + MONTH([Date])   -- 如 202403
WeekNum     = WEEKNUM([Date], 2)                   -- 週一為第一天

-- 日期辅助表（複製日期表，不建立關聯）
日期辅助表 = 日期表
```

**設計解析**：YearMonth 使用整數格式（如 202403）方便排序，比字串 "2024-03" 更安全。

---

### 9.2 庫存類 DAX 公式

```dax
-- 成品期末庫存（取最後一日快照）
成品期末库存金额 =
VAR X = LASTDATE('D日期表'[Date])
RETURN CALCULATE(
    SUMX('F成品库存明细', [结存金额]),
    'D日期表'[Date] = X
)

-- 原料期末庫存（同上）
原料期末库存金额 =
VAR X = LASTDATE('D日期表'[Date])
RETURN CALCULATE(
    SUMX('F原材料库存明细', [结存金额]),
    'D日期表'[Date] = X
)

-- 總庫存
總库存 = [成品期末库存金额] + [原料期末库存金额]

-- 成品期初庫存（用 FIRSTDATE）
成品期初库存金额 =
VAR X = FIRSTDATE('D日期表'[Date])
RETURN CALCULATE(
    SUMX('F成品库存明细', [结存金额]),
    'D日期表'[Date] = X
)
```

**解析**：`LASTDATE` 返回當前篩選上下文中日期表的最後一天，確保期末庫存只取快照而非加總。

```dax
-- 在檔 SKU 數（成品主檔全部）
在档SKU数 = COUNTROWS('D成品档案')

-- 在售 SKU 數（當期有出庫的）
在售SKU数 = COUNTROWS(
    FILTER(VALUES('D成品档案'[存货编码]), [出库金额] > 0)
)

-- 動銷率
动销率 = DIVIDE([在售SKU数], [在档SKU数])

-- 近三月平均庫存
近三月平均库存 = CALCULATE(
    AVERAGEX(VALUES('D日期表'[Date]), [成品期末库存金额]),
    DATESINPERIOD('D日期表'[Date], MAX('D日期表'[Date]), -3, MONTH)
)

-- 近三月在庫 SKU 數（近三月有庫存的）
近三月在库SKU数 = COUNTROWS(
    FILTER(VALUES('D成品档案'[存货编码]), [近三月平均库存] > 0)
)

-- 近三月在庫動銷率（更精準）
在库动销率 = DIVIDE([在售SKU数], [近三月在库SKU数])

-- 近三月未動銷 SKU 數
近三月未动销SKU数 = COUNTROWS(
    FILTER(
        VALUES('D成品档案'[存货编码]),
        CALCULATE(
            [成品出库金额],
            DATESINPERIOD('D日期表'[Date], MAX('D日期表'[Date]), -3, MONTH)
        ) <= 0
    )
)
```

```dax
-- 庫存周轉天數
日均出库金额 = AVERAGEX(VALUES('D日期表'[Date]), [每日出库金额])
月周转天数  = DIVIDE([期末库存余额], [日均出库金额])
```

**解析**：`DATESINPERIOD` 以當前最大日期為基準，向前推 N 個月，是滾動計算的標準寫法。

```dax
-- 原料分類（通用料/專用料/非生產用料）
原料生产成品种类数量 =
VAR x = MAX('原材料档案'[存货编码])
VAR y = FILTER(
    SUMMARIZE('生产成本明细', '生产成本明细'[存货编码], '生产成本明细'[费用编码]),
    '生产成本明细'[费用编码] = x
)
RETURN COUNTROWS(y)

-- SWITCH TRUE 分類
原料类型 =
SWITCH(
    TRUE(),
    [原料生产成品种类数量] > 5,      "通用料",
    [原料生产成品种类数量] > 0,      "专用料",
    "非生产用料"
)
```

**解析**：`SWITCH(TRUE(), ...)` 等同 IF-ELSEIF 串聯，比嵌套 IF 更易讀，且按順序判斷（> 5 先判）。

### 9.3 日期輔助表 + 趨勢計算

```dax
-- 趨勢折線：取近 12 個月，透過 INTERSECT 連結輔助表
母件产量趋势（日期辅助表）=
VAR X = DATESINPERIOD('D日期表'[Date], MAX('D日期表'[Date]), -1, YEAR)
VAR Y = VALUES('D日期辅助表'[Date])
VAR Z = CALCULATE([母件产量], INTERSECT(X, Y))
RETURN Z
```

**解析**：
- `DATESINPERIOD` 計算近 12 個月日期列表（表函數，不含起始邊界日）
- `VALUES('D日期辅助表'[Date])` 取輔助表當前筛選下的日期
- `INTERSECT(X, Y)` 取交集，且繼承第一個參數（X）的篩選關係 → 保留主日期表的數據沿襲
- 結果：折線圖不受切片器選月影響，始終顯示完整 12 個月趨勢

```dax
-- 採購趨勢（含 TREATAS 跨模型篩選）
采购趋势 =
VAR X = DATESINPERIOD('D日期表'[Date], MAX('D日期表'[Date]), -12, MONTH)
VAR Y = VALUES('D日期辅助表'[Date])
RETURN CALCULATE(
    [平均采购价格],
    TREATAS(VALUES('D合并物料表'[存货编码]), '原材料档案'[存货编码]),
    INTERSECT(X, Y)
)
```

**解析**：`TREATAS(tableExpression, column)` 將一個列的值，強制套用到另一個列的篩選上下文，解決無關係路徑的跨表篩選問題。

### 9.4 供應商價格類 DAX 公式

```dax
-- 供應商數量
供应商数量 = DISTINCTCOUNT('F原材料采购到货单列表'[供应商])

-- 採購金額
采购金额 = SUM('F采购到货单'[金额])

-- 採購入庫數量
采购入库数量 = SUM('F采购到货单'[数量])

-- 平均採購單價（原幣，乘以 10000 換算元）
平均采购价格 = DIVIDE([采购金额], [采购入库数量]) * 10000

-- 最低採購單價（先去重再取最小）
最低采购单价 = MINX(
    FILTER(
        SUMMARIZE('F采购到货单', [供应商], [订单号], [存货编码]),
        CALCULATE(AVERAGE([原币单价])) > 0
    ),
    CALCULATE(AVERAGE([原币单价]))
)

-- 最高採購單價
最高采购单价 = MAXX(
    SUMMARIZE('F采购到货单', [供应商], [订单号], [存货编码]),
    CALCULATE(AVERAGE([原币单价]))
)

-- 價格差異率
价格差异率 = DIVIDE([最高采购单价] - [最低采购单价], [最低采购单价])

-- 價格趨勢（最新 vs 平均）
价格趋势 = DIVIDE([最新采购单价] - [平均采购价格], [平均采购价格])

-- 原料消耗排名
原材料消耗排名 = RANKX(
    ALL('D原材料档案'[一级分类]),
    [原料出库金额],
    ,
    DESC
)

-- 消耗佔比
原材料消耗金额占比 = DIVIDE(
    [原料出库金额],
    CALCULATE([原料出库金额], ALL('D原材料档案'))
)

-- 複合顯示標籤（佔比 + 排名）
显示标签 = FORMAT([占比], "0.00%;-0.00%") & REPT(" ", 2) & "排名：" & [排名]
```

### 9.5 採購時效類 DAX 公式

```dax
-- 採購訂單維度表（UNION 去重）
D采购订单号 = FILTER(
    DISTINCT(UNION('F采购订单'[订单编号], 'F到货单'[采购委外订单号])),
    [订单编号] <> BLANK()
)

-- 採購下單日期
采购下单日期 =
VAR X = MAX('D日期表'[Date])
RETURN CALCULATE(
    SELECTEDVALUE('F采购订单'[日期]),
    'D日期表'[Date] <= X
)

-- 採購最早到貨日期
采购到货时间最早 =
VAR X = MAX('D日期表'[Date])
RETURN CALCULATE(
    MIN('F到货单'[日期]),
    'D日期表'[Date] <= X
)

-- 採購最早到貨天數
采购周期最早(天数) = IF(
    [采购到货时间最早] > [采购下单日期],
    DATEDIFF([采购下单日期], [采购到货时间最早], DAY),
    BLANK()
)

-- 平均採購周期（對每張訂單 AVERAGEX）
平均采购周期 = AVERAGEX(
    VALUES('D采购订单号'[订单编号]),
    [采购周期最早(天数)]
)

-- 遲交次數
采购周期晚于计划次数 = COUNTROWS(
    FILTER(
        VALUES('D采购订单号'[订单编号]),
        [采购周期最早(天数)] > [采购周期计划(天数)]
    )
)

-- 早於計劃次數
采购周期早于计划次数 = COUNTROWS(
    FILTER(
        VALUES('D采购订单号'[订单编号]),
        [采购周期最早(天数)] < [采购周期计划(天数)]
    )
)

-- 遲交率
延迟到货率 = DIVIDE([采购周期晚于计划次数], [总下单次数])

-- 平均遲交天數（只計遲交的訂單）
晚于计划采购周期平均天数 =
VAR X = ADDCOLUMNS(
    FILTER(
        VALUES('D采购订单号'[订单编号]),
        [平均采购周期最早与计划平均周期差异] > 0
    ),
    "差异天数", [平均采购周期最早与计划平均周期差异]
)
RETURN AVERAGEX(X, [差异天数])
```

**解析**：`ADDCOLUMNS` 先在 FILTER 後的訂單列表上附加計算欄，再用 AVERAGEX 對其求平均，是「先篩再聚合」的標準模式。

### 9.6 生產成本類 DAX 公式

```dax
-- 母件產量（訂單粒度去重後求和）
母件产量 =
VAR T = SUMMARIZE(
    'F生产成本明细',
    [存货编码], [订单号], [订单行号], [产量]
)
RETURN SUMX(T, [产量])

-- 上期母件產量
母件产量上期 = CALCULATE(
    [母件产量],
    DATEADD('D日期表'[Date], -1, MONTH)
)

-- 子件實際消耗金額
子件实际消耗金额 = SUM('F生产成本明细'[金额])

-- 上期消耗金額
子件上期实际消耗金额 = CALCULATE(
    [子件实际消耗金额],
    DATEADD('D日期表'[Date], -1, MONTH)
)

-- 單位生產成本（固定元為單位，不受切片器影響）
单位生产成本 =
DIVIDE(
    [子件实际消耗金额] * [单位倍率],  -- 萬元→元換算
    [母件产量]
)

-- 生產種類數
生产种类数 = DISTINCTCOUNT('F生产成本明细'[存货编码])

-- 上期生產種類數
生产种类数上期 = CALCULATE(
    [生产种类数],
    DATEADD('D日期表'[Date], -1, MONTH)
)

-- 母件單位（矩陣顯示用）
母件单位 = SELECTEDVALUE('F生产成本明细'[单位])

-- 子件消耗數量
子件消耗数量 = SUM('F生产成本明细'[数量])

-- 子件每單位用量（量差分析）
子件每单位用量 = DIVIDE([子件消耗数量], [母件产量])

-- 子件單位金額（價差分析）
子件单位金额 = DIVIDE([子件实际消耗金额], [子件消耗数量]) * [单位倍率]
```

### 9.7 成本差異類 DAX 公式

```dax
-- 環比差異額（產量）
产量差异额 = [母件产量] - [母件产量上期]

-- 環比差異率（產量）
产量差异率 = DIVIDE([产量差异额], [母件产量上期])

-- 同比產量（YoY，僅適用自產自銷企業）
母件产量同期 = CALCULATE(
    [母件产量],
    SAMEPERIODLASTYEAR('D日期表'[Date])
)

-- 單位生產成本環比差異率
单位成本差异率 = DIVIDE(
    [单位生产成本] - [单位生产成本上期],
    [单位生产成本上期]
)

-- 單位生產成本異動提醒（絕對值，供條件格式用）
单位成本异动提醒 = ABS([单位成本差异率])
```

**條件格式分級規則（旗幟圖標）**：

| 條件 | 圖標 | 說明 |
|------|------|------|
| `ABS(差異率) < 1%` | 無圖標 | 正常誤差 |
| `1% ≤ ABS(差異率) < 2%` | 半填充黃旗 | 輕微異常 |
| `2% ≤ ABS(差異率) < 5%` | 全填充黃旗 | 明顯異常 |
| `ABS(差異率) ≥ 5%` | 紅旗 | 嚴重異常，立即處理 |

## 十、優化方向與新架構設計建議

### 10.1 現有架構優化方向

#### (A) 計算效能優化

| 現有問題 | 優化建議 |
|---------|---------|
| FILTER + SUMMARIZE 巢狀使用 | 改用 CALCULATETABLE + DISTINCT 提升效能 |
| 多次 DATEADD 重複計算 | 建立「基期度量值群」集中管理，其他公式直接引用 |
| MINX / MAXX 在大資料表效率低 | 先在 Power Query 建立「訂單最低單價」欄，再 DAX MIN |
| 日期辅助表每次 INTERSECT | 建立共用「近12月日期集合」計算群組（Calculation Group） |

#### (B) 模型結構優化

```
現有：事實表直接含費用分類欄位
建議：從事實表獨立出「D成本類型表」（已部分實踐）
      → 三級層級：一級(材料/人工/製造) → 二級(物料分類) → 三級(費用名稱)
      → 與事實表通過「費用類型」欄位一對多關聯
```

#### (C) 可維護性優化

1. **度量值分組**：使用 Display Folder 為度量值分類（庫存組、採購組、成本組、差異組）
2. **命名規範**：採用 `[模組.指標]` 格式，如 `[库存.期末余额]`、`[成本.单位差异率]`
3. **參數表**：建立「系統參數表」存放異動閾值（如 1%、2%、5%），方便業務部門動態調整

#### 建議新增模組

**模組一：智能異常雷達（Anomaly Radar）**
- 使用條件格式 + KPI 視覺物件，建立全廠「異常地圖」
- 每個分析維度（庫存/採購/成本）各有異常評分

**模組二：RCA 根因分析鑽取**

### 10.4 DAX 進階設計建議

#### 計算群組（Calculation Groups）應用

```
// 建立「時間比較」計算群組
本期: SELECTEDMEASURE()
上期: CALCULATE(SELECTEDMEASURE(), DATEADD('D日期表'[Date], -1, MONTH))
差異額: [本期] - [上期]
差異率: DIVIDE([本期] - [上期], [上期])
同期: CALCULATE(SELECTEDMEASURE(), SAMEPERIODLASTYEAR('D日期表'[Date]))
```

**優點**：50+ 個度量值無需各自寫上期公式，統一由計算群組管理，維護成本降低 80%。

#### 欄位參數（Field Parameters）應用

```
// 建立「分析維度」欄位參數
分析维度 = {
    ("供应商", NAMEOF('D供应商'[供应商名称]), 0),
    ("物料一级", NAMEOF('D原材料档案'[一级分类]), 1),
    ("物料二级", NAMEOF('D原材料档案'[二级分类]), 2),
    ("产品", NAMEOF('D成品档案'[存货名称]), 3)
}
```

**優點**：使用者自行切換分析維度，一張圖表替代四張圖表，報告頁數減少 40%。

### 10.5 業務視角：三大核心洞察框架

```
庫存分析 → 問：「哪些錢是死錢？」
  → 超期庫存 + 非生產用料 = 需要清理的無效資金

採購分析 → 問：「哪裡花了冤枉錢？」
  → 同品項多供應商價差大 = 議價空間
  → 遲交集中的供應商 = 更換評估名單

成本分析 → 問：「哪條產線/哪個物料最貴？」
  → 量差大 = 製程損耗問題
  → 價差大 = 採購成本問題
  → 單位成本逆規模 = 設備/品質問題
```
